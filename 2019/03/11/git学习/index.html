git学习

1. git指令:

###Git基础指令

(1) git pull：git pull origin master,将origin版本库中的代码更新到本地的master分支

(2) git add: 将当前更改或者新增的文件加入到git索引中，加入到索引中代表记入了git版本历史中，这个是进行提交之前所要做的必要工作，git add com/ibk/txn/service/pubService.java就是将com/ibk/txn/service/pubService.java文件加入到git索引中。

(3) git rm:从当前的工作空间中和索引中删除文件,git rm com/ibk/txn/service/pubService.java,就会删除pubService文件

(4) git commit:提交当前工作空间的修改内容，git commit -m story #3,commit this service,提-m相当于提交信息

(5) git push:将本地commit后的代码更新到远程版本库中，git push origin就是将本地代码更新到名为origin的远程库中。

(6) git log:查看历史日志

(7) git revert:还原一个版本的修改，必须提供具体的版本号:git revert 版本号，(版本号一般都是生成的一个哈希值)

###进阶git指令

(1) git branch：可以进行创建，删除，查询分支, git branch lc_Branch会从当前工作版本中创建一个 lc_Branch的分支，git branch -D lc_Branch就会删除lc_Branch分支，git branch就会列出本地所有git分支

(2) git checkout：Git的checkout有两个作用，其一是在不同的branch之间进行切换，例如’git checkout new_branch’就会切换到new_branch的分支上去；另一个功能是还原代码的作用，例如’git checkout app/model/user.rb’就会将user.rb文件从上一个已提交的版本中更新回来，未提交的内容全部会回滚

(3) git rebase:用于把一个分支的修改合并到当前分支。使用git rebase后，可以让不同前进的分支历史看起来像没有经过合并(merge commit)一样，

(4) git reset:将git版本回滚到指定的版本号:比如，有3次提交的版本A-C,其中B版本的版本的版本号(哈希值)。如果使用该指令git reset B版本号，此时git中就只会剩下A-B的版本

(5) git stash：将未提交的代码存入git工作栈中。需要时再应用回来

(6) git config:可以新增，更改git的设置,如git config branch.master.remote origin就是将master的远程版本库设置为别名为origin的版本库

(7) git tag:为某一个版本打上标签，如:git tag study_version 版本哈希值，下次就可以根据study_version标签进行查看该版本，而不用使用哈希值了

3. 创建冲突文件:

(1) 首先你得创建一个远程仓库:在本地E盘中我新建了一个git文件夹，利用命令git init初始化git仓库，进入到之前创建的本地仓库git-study,然后通过命令git remote add origin E:\git\将本地仓库与远程仓库关联。

此时，因为创建的不是裸仓库，所以我们得进行相应的配置。git config –global receive.denyCurrentBranch ignore

也可以直接修改配置文件：.git/config

在文件中添加以下内容即可

[receive]

denyCurrentBranch = ignore

配置完成后就能将代码push到远程库中了。
