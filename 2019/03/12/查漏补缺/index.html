每日一查

发表于 2019-03-12 | 分类于 java

  1. 二维数组

    二维数组的length属性取得是二维数组的行数

  1. 如果要将String字符串的某某字符给替换，首先考虑下charAt()方法。重新做一遍，将空格替换成8
  1. StringBuffer是线程安全的，StringBuilder是线程不安全的
  1. Static中在未实例化对象时，只能使用同为static的方法或属性

  1. 抽象类和接口不能实例化

    1. 抽象类:abstract只能用public,abstract和final进行修饰，abstract类可以实现超类的所有抽象方法，也可以实现部分超类的抽象方法，但是非抽象类必须实现超类(超类是抽象类)所有抽象方法才行.abstract可以修饰类或方法，但不能修饰属性，抽象方法没有方法体，即没有大括号，

    2. 接口:接口只能用 public和abstract进行修饰。

      接口可以实现多实现，接口中定义属性为常量

  1. Spring有AOP,但是自身未提供日志系统

  1. 类之间的依赖关系通过配置文件进行建立，而不是通过ORM建立的

  2. Integer类型的数据，如果在-128-127之间时，创建的对象会放到常量池中，如果new Integer时，会在堆中重新开辟一个空间存放，此时会与正常的Integer对象的引用值不同

  3. 一个类中可以有多个main方法，但是public static void main(String[] args){}只能有一个。

  4. 子类可以继承父类的所有成员，只是私有成员无法使用而已

  5. 继承要保证3个原则。3同一小一大

    3同:返回类型相同，方法名相同，参数列表相同

    一小:子类方法throw异常必须比父类小或者相等

    一大:子类方法的访问权限修饰符必须大于等于父类的权限修饰符

  6. 补码,反码和原码:

    https://www.nowcoder.com/questionTerminal/a568949ffbfc4a20977a2dab786032dd

    ~10为10全部取反得到的结果

    ~10的原码为11111111 11111111 11111111 11110101（这个就是补码）

    ~10的反码为10000000 00000000 00000000 00001010（最高位符号位，不变，其余位取反）

    ~10的补码为10000000 00000000 00000000 00001011（负数的补码=反码+1)

    也就是说，-10=~10+1

    正数的补码，反码，原码相同

    负数的反码为原码除了符号外，按位取反，

    负数的补码为反码加1

  7. 父类用private修饰的方法对于子类是不可见的，比如private final void fct()对于子类来说不可见，因此如果子类中也有private final void fct()就不会报错，因为这里是2个方法，但是如果将父类中的private修改成public或protected则会出错

  8. switch(){}分支语句中如果其中一个分支没有break;终止，则会继续进行下去。在jdk1.7以后，switch(x)中x可以为字符串 ，在1.7以前x只能为byte,short,char,int

    switch(){}分支语句中，switch第一次进入case语句的地方为()中的数字

  9. ceil,floor,round的区别：

    ceil，大于等于该值的最小一个整数

    floor,小鱼等于该值的最大一个整数

    round,离该值最近的一个整数

  10. ThreadLocal(线程变量):

    1、线程变量是为每一个线程创建一个本地副本，可以实现线程内部的资源共享，而线程外部是互斥的，

    2、ThreadLocal是以空间换时间，而线程同步是以时间换空间。

    3、在ThreadLocal类中有一个Map，用于存储每一个线程的变量副本

  11. ==的优先级要高于三目运算符。

  12. 引用传递中，如果对引用进行修改，不会影响原对象，只有对引用内部的值进行修改，才会影响原对象。

  13. private方法在本类中可以使用，无论用什么方法进行调用。。

  14. java属于解释性的语言，在运行时才会执行指令，且低版本的jvm不能运行高版本的java代码

  15. ==在比较基本数据类型时，比较的是两边的内容，float a = 2.0f,long b = 2l;a==b为 true

  16. 整数型默认为int型，小数型默认为double型

  17. c++知识点集合:

    (1). 公有继承的公有成员，可以直接访问

    (2). 公有继承的私有成员，不可以访问

    (3). 公有继承的保护成员，不可以直接访问

    (4). 私有继承的公有成员，不可以直接访问

  18. Math.round()代表四舍五入，Math.round(11.5)=12;

    Math.ceil代表天花板:Math.ceil(11.2)=12.0;

    Math.floor代表地板:Math.floor(11.8)=11.0;

  19. 使用super()或者this()方法是必须放在构造函数的第一行，this()中存在调用super(),所以this()和super()不能出现在同一个构造函数中。

  20. 接口可以多继承，类只能单继承。

  21. 例如: i = i++;这种表达式执行完后，i为多少，实际上就是为i的初始化值，因为相当于一开始将i设置到一个temp临时变量中。

  22. abstract抽象类和接口都不能实例化。但是抽象类有构造方法，接口没有构造方法

    尽量多使用接口，少使用抽象类继承，因为继承耦合度太高

  23. 方法名可以和类名一致

  24. Throwable为Exception和Error的基类，Exception包含RuntimeException和非运行时异常

  25. web中会话跟踪技术，使得无状态的Http协议，变成带有状态编程。也就是在进行通信时可以携带客户端的状态。存在以下4中会话跟踪技术:

    • 隐藏表单域:
    • url重写:在URL后面拼接参数
    • Cookie:服务器使用SET-Cookie头标将它作为HTTP响应的一部分传送到客户端，客户端被请求保存Cookie值。
    • Session:使用setAttribute(String str, Object obj)带参数

  26. 使用Condition代替传统的Object中的wait/notify/notifyAll方法，利用await()、signal()这种方式会更加高效
